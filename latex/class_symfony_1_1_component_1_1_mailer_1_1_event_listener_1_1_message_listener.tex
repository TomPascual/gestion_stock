static Symfony\textbackslash{}\+Component\textbackslash{}\+Mailer\textbackslash{}\+Event\+Listener\textbackslash{}\+Message\+Listener\+::get\+Subscribed\+Events (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [static]}}

Returns an array of event names this subscriber wants to listen to.

The array keys are event names and the value can be\+:


\begin{DoxyItemize}
\item The method name to call (priority defaults to 0)
\item An array composed of the method name to call and the priority
\item An array of arrays composed of the method names to call and respective priorities, or 0 if unset
\end{DoxyItemize}

For instance\+:


\begin{DoxyItemize}
\item \mbox{[}\textquotesingle{}event\+Name\textquotesingle{} =\texorpdfstring{$>$}{>} \textquotesingle{}method\+Name\textquotesingle{}\mbox{]}
\item \mbox{[}\textquotesingle{}event\+Name\textquotesingle{} =\texorpdfstring{$>$}{>} \mbox{[}\textquotesingle{}method\+Name\textquotesingle{}, \$priority\mbox{]}\mbox{]}
\item \mbox{[}\textquotesingle{}event\+Name\textquotesingle{} =\texorpdfstring{$>$}{>} \mbox{[}\mbox{[}\textquotesingle{}method\+Name1\textquotesingle{}, \$priority\mbox{]}, \mbox{[}\textquotesingle{}method\+Name2\textquotesingle{}\mbox{]}\mbox{]}\mbox{]}
\end{DoxyItemize}

The code must not depend on runtime state as it will only be called at compile time. All logic depending on runtime state must be put into the individual methods handling the events.
