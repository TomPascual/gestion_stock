{\footnotesize\ttfamily \label{class_ramsey_1_1_uuid_1_1_rfc4122_1_1_fields_aab123e9317e495fbe5166bf141a50b95} 
Ramsey\textbackslash{}\+Uuid\textbackslash{}\+Rfc4122\textbackslash{}\+Fields\+::get\+Timestamp (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})}

Returns the full 60-\/bit timestamp, without the version

For version 2 UUIDs, the time\+\_\+low field is the local identifier and should not be returned as part of the time. For this reason, we set the bottom 32 bits of the timestamp to 0\textquotesingle{}s. As a result, there is some loss of fidelity of the timestamp, for version 2 UUIDs. The timestamp can be off by a range of 0 to 429.\+4967295 seconds (or 7 minutes, 9 seconds, and 496730 microseconds).

For version 6 UUIDs, the timestamp order is reversed from the typical RFC 4122 order (the time bits are in the correct bit order, so that it is monotonically increasing). In returning the timestamp value, we put the bits in the order\+: time\+\_\+low + time\+\_\+mid + time\+\_\+hi.
