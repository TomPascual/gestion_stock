\doxysubsection{Detailed Description}
A Controller\+Resolver\+Interface implementation knows how to determine the controller to execute based on a Request object.

A Controller can be any valid PHP callable.

Symfony\textbackslash{}\+Component\textbackslash{}\+Http\+Kernel\textbackslash{}\+Controller\textbackslash{}\+Controller\+Resolver\+Interface\+::get\+Controller (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_symfony_1_1_component_1_1_http_foundation_1_1_request}{Request}}}]{\$request}{}\end{DoxyParamCaption})}

Returns the Controller instance associated with a Request.

As several resolvers can exist for a single application, a resolver must return false when it is not able to determine the controller.

The resolver must only throw an exception when it should be able to load a controller but cannot because of some errors made by the developer.

\begin{DoxyReturn}{Returns}
callable\texorpdfstring{$\vert$}{|}false A PHP callable representing the Controller, or false if this resolver is not able to determine the controller
